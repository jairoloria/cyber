Advanced Network Security Lab

Man-in-the-Middle Operations and analysis

A technical walk-through of ARP Cache Poisoning, DNS Hijacking, and Forensic Analysis.

01. Lab Topology & Setup

The environment consists of two virtual machines connected to a custom VirtualBox NAT Network with Promiscuous Mode enabled.

Attacker Node:
OS: Kali Linux
IP: 10.0.2.4
Target Node:
OS: Windows / Linux Mint
IP: 10.0.2.15
Virtual Gateway:
DHCP Router
IP: 10.0.2.1

02. ARP Spoofing Phase

The objective is to redirect the traffic flow. By sending forged ARP replies, the attacker forces the victim to update its ARP table with the attacker's MAC address for the Gateway IP.

Open a terminal in Kali linux and enter the following code

# Terminal 1: Poisoning Victim's Cache
sudo arpspoof -i eth0 -t 10.0.2.15 10.0.2.1

# Terminal 2: Poisoning Gateway's Cache
sudo arpspoof -i eth0 -t 10.0.2.1 10.0.2.15
Figure 1: Concurrent ARP spoofing terminals maintaining the MitM position.

03. DNS Hijacking Phase

Once the MitM position is established, we intercept DNS queries. The dnsspoof tool monitors for specific domain requests and replies with a spoofed IP before the real DNS server can respond.

1. Create in Desktop a txt file with the DNS and ip in the following order

# Configuration file (fake_hosts.txt)


# Execution
nano fake_hosts.txt

8.8.8.8 www.facebook.com # Save with Cntrl+0


sudo dnsspoof -i eth0 -f ~/Desktop/fake_hosts.txt
sudo dnsspoof -i eth0 -f fakehosts.txt

04. Forensic Analysis (Wireshark)

Analyzing the captured traffic confirms the attack success through protocol anomalies.

  • ARP Detection: Wireshark flags "Duplicate IP address configured" warnings (two MACs for one IP).
  • DNS Analysis: Identifying the forged "Standard Query Response" originating from the attacker's MAC.
Figure 3: Wireshark capture showing forged DNS packets and ARP conflicts.

05. Security & Mitigations

To defend against these attacks, we implement the following industry-standard protections:

  • Dynamic ARP Inspection (DAI): Prevents ARP poisoning by validating ARP packets against a DHCP snooping database on the switch.
  • DNS over HTTPS (DoH): Encrypts DNS requests, preventing attackers from reading or spoofing the responses.
  • Static Neighbors: For critical servers, manually binding MAC addresses to IPs via netsh or ip neighbor.
