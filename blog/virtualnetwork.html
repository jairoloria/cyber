Building a Secure Virtual Environment

Step-by-step guide to NAT Network creation and VM isolation in VirtualBox.

Step 1: Creating the Global NAT Network

Before connecting virtual machines, we must define a virtual switch that allows internal communication while providing controlled internet access.

  • Go to File > Network
  • Click on the NAT Networks tab.
  • Click Create to generate a new interface.
  • Configure the following parameters:
Parameter Value
Network Name Lab1
Network CIDR 10.0.2.0/24
DHCP Support Enabled (Checked)

Step 2: Assigning VMs to the Network

Each machine (Attacker and Victim) needs to be manually linked to the newly created virtual segment.

  • Select your VM (Kali Linux / Linux Mint) and click on Settings.
  • Navigate to the Network section.
  • Set "Attached to" to NAT Network.
  • Select Lab1 from the Name dropdown menu.

Step 3: Enabling Promiscuous Mode

To perform packet sniffing and MITM attacks, the virtual network adapter must be able to "see" all traffic on the segment, not just traffic directed to its own MAC address.

  • Expand the Advanced section in the Network settings.
  • Locate Promiscuous Mode.
  • Change the setting to Allow All.

Step 4: Connectivity Verification

Ensure both machines can communicate and reach the internet through the virtual gateway.

  • Boot both VMs and verify IP addresses using ip address.
  • Test the gateway connection: ping 10.0.2.1.
  • The environment is now ready for security testing.
